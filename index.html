The aim of the Scar project is to extract the Sun Cloud API specification details and description from the HTML code of the Kenai wiki pages, into a more accessible format, e.g. JSON or YAML. How these details and descriptions are to be stored and rendered is being determined. In doing so the Scar Project should facilitate developing and employing "The Sun Cloud API" by increasing

Getting Started

If you'd like to contribute to Scar, please fork hedgehog/scar on GitHub, and push up a topic branch for each feature you add or bug you fix. Then create a Github issue, link to the topic branch and explain what the change does. This allows discussion and merging of each change separately. To contribute:

  1. Fork the scar project using your github account.
  2. Clone your git fork, e.g. to get the API version tagged vm.n.x.y 
    git clone git@github.com:<gh-user>/scar.git /usr/src/scar
pushd /usr/src/scar
git remote add upstream git://github.com/hedgehog/scar.git
git fetch upstream
git checkout -b your_branch vm.n.x.y
  3. To make changes: edit/create features and specs commit, pull, push etc. to your local then github repository. See here for details on how to pull from upstream. Send pull requests, ideally pulling from a topic branch that results in a fast forward merge with upstream.
    1. Ensure changes pass all the features and specs: 
      pushd <repo-root>
cucumber ./features/scar.feature
etc.
  4. Publish the web page presentation of your fork's contents. Please read the rake task contents before executing.
    1. Activate gh-pages for your fork of the scar project. See details here.
    2. To setup the local gh-pages repository 
      pushd <repo-root>
rake site:github:setup
popd
    3. To migrate the local copy of the nanoc3 compiled web pages to your github project pages: 
      pushd <repo-root>
rake site:github:migrate
popd

Scar Project Objectives

Agility: Provide snapshots of Sun's Cloud API. Example To grab version 0.n 

git clone git://github.com/hedgehog/scar.git
git checkout -b your_branch v0.n
Github is not configured to respond to git archive.... Consequently, if you don't want the whole repo, use 
curl -O -L http://github.com/hedgehog/scar/downloads/...

Transparency: Make transparent, and accessible, any ideas others explore or errors they correct. See here.

Productivity: Generate documentation from the API specification. Projects implementing Sun's Cloud API can share a common documentation source and generate (possibly filtered) documentation that is extracted from the specification, and automatically evolves as the specification is corrected or extended.

Version numbering: The Scar project's version number is made up of four parts: m.n.x.y. The first two numbers, m and n track Sun's API version numbers. The second two, x and y, are specific to Scar's development. Increments in x will indicate when changes in 'The Sun API' specification (currently wiki pages) have been incorporated into the Scar project. Increments in y will indicate Scar specific revisions, which can be minor changes, e.g. typographical errors, or major, e.g. refactoring. The release notes should give some details of the changes.
